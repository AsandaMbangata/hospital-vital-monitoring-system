# Reflection

Working on this assignment highlighted the importance of selecting and adapting project management tools to suit specific system requirements. One of the main challenges was choosing the most suitable GitHub template. Although multiple templates were available, it required careful evaluation to determine which one best aligned with Agile workflows and the needs of the Hospital Vital Monitoring System.

Customizing the Kanban board also presented difficulties. Deciding which columns to include, such as “Testing” and “Backlog,” required thinking about how tasks realistically move through development stages. Initially, I struggled to balance simplicity with completeness, but adding these columns helped create a more accurate and practical workflow.

Another challenge was comparing GitHub Projects with tools like Trello and Jira. GitHub integrates well with code and issues, but it is less intuitive than Trello, which is more visual and user-friendly. Jira provides advanced features but is more complex. This comparison helped me better understand the strengths and limitations of each tool.

Working individually also made decision-making more difficult, as I had to rely on my own understanding without team input. This pushed me to think critically and make informed choices about workflow design.

Overall, this assignment improved my understanding of Kanban boards and Agile practices, showing how proper task visualization and organization can support efficient development.
