# Template Analysis and Selection

## Comparison of GitHub Project Templates

| Template | Columns / Workflow | Automation Features | Agile Suitability |
|----------|------------------|--------------------|-------------------|
| Basic Kanban | To Do, In Progress, Done | No automation | Good for simple task tracking but requires manual updates |
| Automated Kanban | To Do, In Progress, Done | Automatically moves issues based on status | Highly suitable for Agile workflows and sprint tracking |
| Bug Triage | Needs Review, In Progress, Done | Auto-labeling and issue sorting | Best for handling bugs, not full development workflow |
| Team Planning | Backlog, Ready, In Progress, Done | Supports planning and prioritization | Good for structured Agile planning but more complex |

---

## Selected Template: Automated Kanban

### Justification

The Automated Kanban template was selected because it aligns well with Agile principles and the needs of the Hospital Vital Monitoring System.

- It supports **continuous workflow tracking**
- It automatically moves issues across columns, reducing manual effort
- It integrates well with **GitHub Issues created in Assignment 6**
- It allows clear visibility of task progress from backlog to completion

Compared to other templates, Automated Kanban is the most suitable because it balances simplicity and functionality while supporting Agile development practices such as iterative delivery and task tracking.
